Prom.Exporter

exporter 采集监控数据通过prom 监控规范对外提供数据的组件。通过监控对象提供的监控相关接口获取监控数据。通过以下方式对外提供：

  1. HTTP/HTTPS
  2. TCP: MySQL exporter 通过开放的监控相关表获取监控指标
  3. The local file: Node exporter 通过读取proc文件
  4. 标准协议方式

官方提供了各种语言的依赖

Node_Exporter

从proc文件系统中获取指标数据

# 读取/proc/meminfo
file, err := os.Open(procFilePath("meminfo"))

Redis_Exporter

3.x + 版本原生支持集群部署，通过设置环境变量REDIS_ADDR 和REDIS_PASSWORD 连接Redis来获取监控数据，多个Redis节点，通过REDIS_FILE指定一个文件，该文件定义多个Redis节点。

通过Redis原生的INFO ALL 命令获取Redis的所有指标

# 通过doRedisCmd发送INFO ALL命令
infoAll, err := redis.String(doRedisCmd(c, "INFO", "ALL"))

Mysql_Exporter

官方建议使用5.5 +，通过设置环境变量DATA_SOURCE_NAME指定MySQL地址，读取MySQL的性能数据，读取表需要用户权限(进程表，库，主从位置，表)。

#
CREATE USER 'exporter'@'localhost' IDENTIFIED BY 'passwd' WITH MAX_USER_CONECTIONS 3;
CRANT PROCESS,REPLICATION CLIENT,SELECT ON *.* TO 'exporter'@'localhost';

系统信息存放infomation_schema，性能信息存放performance_schema(5.7 默认开启 performance_schema=ON)

GO (client_golang)

exporter中通常有多个协程共同修改某个指标，如何保证数据一致性？

client_goland中引入原子变量，记录指标变化

# 变量累加操作 Add()
func (g *gauge) Add(val float64) {
  for {
    oldBits := atomic.LoadUint64(%g.valBits)
    newBits := math.Float64bits(math.Float64frombits(oldBits) + val)
    if atomic.CompareAndSwapUint64(&g.valBits, oldBits, newBits) {
      return
    }
  }
}
