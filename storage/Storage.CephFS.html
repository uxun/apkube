CephFS

1.ENV Prepare

  1. ceph存储集群
  2. Kernel Version > 4.x (需要使用旧的内核，应该使用fuse客户端，不使用内核客户端)
  3. CephFS BEST PRACTICES

2.Deploy

CephFS 需要配置metadata server

至少需要两个RADOS池，一个元数据(ssd)服务器，一个用于数据。

用于创建文件系统的数据池是“默认”数据池和用于存储所有inode回溯信息的位置，用于硬链接管理和灾难恢复

需要了解下擦除编码池(ecpool)，对象在多个磁盘复制，默认支持丢失两个OSD，简单点说可以做到2TB存1T数据。

# create a metadata server
$ ceph-deploy mds create hadoop-s02-235

# create file system
$ ceph osd pool create cephfs_data 128
$ ceph osd pool create cephfs_metadata 128
# new fs
$ ceph fs new cephfs cephfs_metadata cephfs_data

# 配置密钥
$ cat /etc/ceph/admin.secret
AQB7LVBb9xTNFBAAjZfXOChO685MHBby2oADPQ==

# 配置文件
$ cat /etc/ceph/ceph.client.admin.keyring
[client.admin]
    key = AQB7LVBb9xTNFBAAjZfXOChO685MHBby2oADPQ==
    caps mds = "allow *"
    caps mon = "allow *"
    caps osd = "allow *"
获取 Ceph ceph.client.admin.keyring 并生成 secret key
$ ceph auth get-key client.admin |base64

3.Mount

内核驱动程序 or 用户空间(FUSE)

将CephFS挂载为内核驱动 

$ mkdir /mnt/cephfs
$ mount -t ceph {ip-address-of-monitor}:6789:/ /mnt/cephfs

# Ceph存储集群默认使用身份验证。指定用户name和secretfile 
$ mount -t ceph 192.168.1.235:6789:/ /mnt/cephfs -o name=admin,secretfile=/etc/ceph/admin.secret

将CephFS挂载为用户空间(FUSE)中的文件系统

# install
$ yum install -y ceph-fuse
$ mkdir ~/mycephfs
$ ceph-fuse -m {ip-address-of-monitor}:6789 ~/mycephfs

# 指定密钥验证
$ ceph-fuse -k ./ceph.client.admin.keyring -m 192.168.1.235:6789 ~/mycephfs

# 卸载
$ fusermount -u ~/mycephfs

fstab

cat /etc/fstab
#######
192.168.1.235:6789:/  /mnt/cephfs  ceph  name=admin,secretfile=/etc/ceph/admin.secret,noatime,_netdev  0  2
