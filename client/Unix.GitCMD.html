Git Command

branch(分支)

# new branch
$ git branch gh-pages
# 切换分支操作(创建+切换)
$ git checkout -b gh-pages

# push 
$ git push origin gh-pages
# 映射本地目录origin目录
$ git push origin gh-pages:wiki

# 追踪远端分支,生成用于工作的本地分支
# 推荐使用 --track 
$ git checkout --track origin/gh-pages
# default
$ git checkout -b gh-pages origin/gh-pages

Delete

# Delete branch (bug001,idea,hotfix,使用后delete) 
$ git branch -d gh-pages 
# 强制
$ git branch -D gh-pages
# delete origin branch
$ git push origin --delete gh-pages

# 切换分支回退版本
$ git reset --hard 版本号的sha1

Check

# 当前分支提交历史
$ git log --oneline --decorate

# 所有分支的提交历史
$ git log --oneline --decorate --graph --all

# check branch
$ git branch -v

# 查看合并，未合并的分支
$ git branch --merged
$ git branch --no-merged

# 远程分支列表
$ git ls-remote

merge

# 切换master 分支，将gh-pages合并
$ git checkout master
$ git merge gh-pages

Rename

# Git branch rename
$ git branch -m oldName newName
# 删除远程分支
$ git push --delete origin oldName
# 上传新命名的localhost分支
$ git push origin newName
# 修改后的localhost分支关联远程分支
$git branch --set-upstream-to origin/newName

Git Fork Rsync

Quick-operating (cd github/project)

$ git fetch upstream
$ git checkout master
$ git merge upstream/master

The detailed process

1.Fork project

$ git clone https://github.com/uxun/kubeasz.git
# 查看fork项目
$ git remote -v

2.设置同步的上游仓库

$ git remote add upstream https://github.com/gjmzj/kubeasz.git
# 验证
$ git remote -v
origin  https://github.com/uxun/kubes.git (fetch)
origin  https://github.com/uxun/kubes.git (push)
upstream    https://github.com/gjmzj/kubeasz.git (fetch)
upstream    https://github.com/gjmzj/kubeasz.git (push)

3.将上游仓库commits内容存到本地upstream/master

$ git fetch upstream

4.切换项目分支 master 上

$ git checkout master
Already on 'master'
Your branch is up to date with 'origin/master'.

5.将upstream/master 改动的内容合并到本地master 

$ git merge upstream/master
