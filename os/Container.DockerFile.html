Docker Dockerfile

Dockerfile Instructions

docker build (RUN) —> docker run (CMD)

格式参考：
<command>:创建为shell子进程。 —> e.g: /bin/sh -c
["executable", "param1", "param2"]: 内核直接创建，不会正常的shell处理。—> e.g:不会变量替换

RUN 定义：基于Docker file 构建镜像时运行的命令

  • RUN <command>（shell，该命令在shell中运行，默认情况下/bin/sh -c在Linux）
  • RUN ["executable", "param1", "param2"]（exec:执行形式）

CMD 定义：镜像启动默认容器时默认运行的命令(最后一个CMD有效)

  • CMD ["executable","param1","param2"]（执行形式，这是首选形式）
  • CMD ["param1","param2"]（作为ENTRYPOINT的默认参数）
  • CMD command param1 param2
01.FROM:            Dockerfile 文件第一个非注释行，基准镜像。 e.g: FROM busybox:latest
02.MAINTAINER:      作者(depreacted)。 e.g: MAINTAINER "name<name@gmail.com>
03.LABEL:           指定元数据。 e.g: LABEL maintainer
04.COPY:            多个src，dest结尾必须为/。
05.ADD:             支持URL路径和TAR文件(local.tar将自动拽展开，URL.tar不会自动展开 )
06.WORKDIR:         指定Dockerfile工作目录
07.VOLUME:          挂载目录<mountpoint> or ["<mountpoint>"]
08.EXPOSE:          监听端口<port>[/<protocol>] e.g: 121/udp 121/tcp
09.ENV:             定义环境变量。定义格式key=valume，引用$key ${key:-/data/web/html/}
10.RUN:             构建镜像文件过程中运行。RUN<command> or RUN["<executable>","<>"]
11.CMD:             容器启动默认执行体，多个CMD最后一个生效。
12.ENTRYPOINT       定义容器启动以后的执行体。强制--entrypoint
13.USER             用于指定运行镜像时或Dockerfile中指令时的用户名或者UID
14.HEALTHCHECK      健康检查
15.SHELL            定义默认shell
16.STOPSIGNAL       定义信号(STOPSIGNAL signal) e.g:STOPSIGNAL 9
17.ARG              定义参数：builder中使用ARG <name>[=<default value>]
18.ONBUILD          Dockerfile中定义触发器，引用此baseimage时，触发生成

Example：

# 01.FROM Description 
FROM busybox:latest

# 02.MAINTAINER: Uxun<uxunuxun@gmail.com>
# 03.LABEL maintainer="Uxun<uxunuxun@gmail.com>"

# 04.COPY
# 多个src，dest结尾必须为/。e.g: COPY<src>...<dest> or 空白字符使用COPY["<src>""<dest>"]
COPY index.html /data/web/html/
COPY yum.repos.d /etc/yum.repos.d/

# 05.ADD
# 支持URL下载的tar文件，不会自动解压
ADD http://xxx.tar.gz /usr/local/src/
# local.tar文件将会自动展开
ADD xxx.tar.gz /usr/local/src/

# 06.WORKDIR
# workhome
WORKDIR /usr/local/

# 07.VOLUME
VOLUME /data/mysql
VOLUME ["/var/www", "/var/log/apache2", "/etc/apache2"]

# 08.EXPOSE
EXPOSE 80/tcp
EXPOSE 80/udp

# 09.ENV
ENV DOC_ROOT=/data/web/html \
    WEB_SERVER_PACKAGE="nginx-1.15.2"

# 10.RUN
# 1.RUN<command>通常是一个shell命令，且以"/bin/sh -c"(shell子进程)来运行，进程PID不为1，不能接收Unix信号，使用docker stop <container> 命令停止容器时，此进程接收不到SIGTERM信号
RUN cd /usr/local/src && \
    tar xf nginx-1.15.2.tar.gz 
# 2.RUN ["<executable>","<param1>","<param2>"]。这种格式中的参<executable>为需要运行的命令，后面的<paramN>为传递给命令的参数和选项。此格式指定的命令不会以"/bin/sh -c"来发起(直接由内核创建)，因此常见的shell操作入变量替换及通配符替换将不会进行。
# 由内核创建
RUN ["<executable>","<param1>","<param2>"] 
# 由shell创建
RUN ["/bin/bash","-c","<executable>","paramN"]

# 11.CMD三种用法
# 1.以shell子进程运行CMD，id不为1。shell格式底层会调用/bin/sh -c <command>。
CMD /bin/httpd -f -h ${WEB_DOC_ROOT}

# 2.不以shell子进程运行，进程id为1。可接收传递参数。
CMD ["/bin/sh","-c","/bin/httpd","-f","-h ${WEB_DOC_ROOT}"]

# 3.给ENTRYPOINT传参 CMD ["<>","<>"]
CMD ["/bin/httpd","-f","-h ${WEB_DOC_ROOT}"]
ENTRYPOINT ["/bin/sh","-c"]

# 12.ENTRYPOINT
# exec: 
ENTRYPOINT ["executable", "param1", "param2"] (exec form, preferred)
# shell: /bin/sh -c command
ENTRYPOINT command param1 param2 (shell form)

# 13.USER
USER <user>[:<group>] or <UID>[:<GID>]

# 14.HEALTHCHECK 健康检测
HEALTHCHECK --start-period=3s CMD wget -O - -q http://${IP:-0.0.0.0}:${PORT:-80}/

# ARG 
ARG author="name <name@gmail.com>"
LABEL maintainer="${author}"

# tip: CentOS中ps top 进程无区别，待跟进。

CMD ["top"]
