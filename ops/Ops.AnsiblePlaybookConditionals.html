条件判断

==：两个对象是否相等，相等为真
!=：两个对象是否不相等，不相等为真
> | <：值比较，左>右，则为真 | 值比较，左<右，则为真
>= | <=：值比较，左>=右，则为真 | 值比较，左<=右，则为真
and：逻辑与，左与右同时为真，则返回真
or：逻辑或，左或右，一个为真，则返回真
not：对一个操作取反
()：组合，将一组操作体为一

变量判断

defined：判断变量是否定义，已经定义返回真
undefind：判断变量是否定义，未定义返回真
none：变量已经定义，但值为空，返回真

执行结果判断

success,succeeded：判断任务执行状态，成功为真
failure, failed：失败为真
change, changed：任务状态为changed为真
skip, skipped：当任务没有满足条件，而被跳过执行时，则返回真

路径判断

判断是针对ansible主机中的路径，与目标主机无关
file：判断路径是否是一个文件，是则为真
directory：判断路径是否是一个目录，是则为真
link：判断路径是否是一个软链接，是则为真
mount：判断路径是否是一个挂载点，是则为真
exists：判断路径是否存在，存在则为真

字符串判断

lower：判断包含字符的字符串中的字符是否是全小写，全小写为真
upper：判断包含字符的字符串中的字符是否是全大写，全大写为真

整除判断

even：数值为偶数，为真
odd：数值为奇数，为真

other

subset：判断list是不是另一个list的子集，是为真
superset：判断list是不是另一个list的父集，是为真
string：判断对象是否是一个字符串
number：判断对象是否是一个数字

when

如果后续有判断的话需要添加 ignore_errors: ture ，忽略错误继续执行

# ansible 的运行方式，当某个task 错误时候，不会继续执行。

---
- hosts: one
  remote_user: root
  tasks:
  - debug:
     msg: "hello centos 7"
    when: ansible_distribution == "CentOS" and ansible_distribution_major_version == "7"

# ingore_errors
---
- hosts: one
  remote_user: root
  tasks:
  - name: task 1 ls file
    shell: "ls /data/hello"
    register: ls_reg
    ignore_errors: true
  - name: task 2 ok
    debug:
      msg: "OK"
    when: ls_reg.rc == 0
  - name: task 3 no
    debug:
      msg: "no"
    when: ls_reg.rc != 0

  - debug:

  - debug:

    when: testonemsg is change
  - debug:
     msg: "skip"
    when: testonemsg is skip

block

将多个任务整合为一个"块"，对这个块进行条件判断，成立则执行"块"中的所有任务

rescue

救援，当block中任务失败后，会执行rescue中的任务进行补救

always

不管block任务如何，always中的任务都会被执行

E.g {block, rescue,always}

# test block，rescue，always
---
- hosts: one
  remote_user: root
  tasks:
  - block: 
     - shell: "ls /ha"
    rescue:
     - debug:
        msg: "this is error"
    always:
     - debug:
        msg: "hello world"

fail

用来"执行失败"的模块，当fail模块执行后，playbook就会认为有任务失败，终止运行

通常与when结合使用

failed_when：条件成立，将对应任务执行状态设置为失败

changed_when：条件成立，将对应任务执行状态设置为changed

---
- hosts: one
  remote_user: root
  tasks:
  - shell: echo "This is test"
    register: return_value
  - fail:
     msg: 'test is error'
    when: "'test' in return_value.stdout"
  - debug:
     msg: "Stop this test"
